Instrument Y The Instrument component block contains all the fields commonly used to describe a security or instrument. Typically the data elements in this component block are considered the static data of a security, data that may be commonly found in a security master database. The Instrument component block can be used to describe any asset type supported by FIX.

537 QuoteType N Identifies the type of quote.
With indicative quotes, the LP has a "last look" option and the right to reject. The LP will send an Execution Report in response to an Order.
A tradeable quote will result directly in a trade without a last look option. LP will receive an Execution Report directly (no Order).

OrderQtyData Y The OrderQtyData component block contains the fields commonly used for indicating the amount or quantity of an order. Note that when this component block is marked as "required" in a message either one of these three fields must be used to identify the amount: OrderQty, CashOrderQty or OrderPercent (in the case of CIV).
38 OrderQty Y Quantity ordered. This represents the number of shares for equities or par, face or nominal value for FI instruments.
(Prior to FIX 4.2 this field was of type int)
63 SettlType N Indicates order settlement period. If present, SettlDate (64) overrides this field. If both SettlType (63) and SettDate (64) are omitted, the default for SettlType (63) is 0 (Regular)
Regular is defined as the default settlement period for the particular security.
Additionally the following patterns may be uses as well as enum values
Dx = FX tenor expression for "days", e.g. "D5", where "x" is any integer > 0
Wx = FX tenor expression for "weeks", e.g. "W13", where "x" is any integer > 0
Noted that tenors expressed using Dx, and Wx values do not denote business days, but calendar days.
64 SettlDate N Specific date of trade settlement (SettlementDate) in YYYYMMDD format.
If present, this field overrides SettlType (63). This field is required if the value of SettlType (63) is B (Broken Date).
193 SettlDate2 N SettlDate (64) of the far leg part of an F/X swap order.
192 OrderQty2 N OrderQty (38) of the future part of a F/X swap order.
15 Currency N Identifies currency used for Qty. Absence of this field is interpreted as the default (base or left side of the pair). It is recommended that systems provide this currency value to avoid confusion.
Can be used to get all Currency Pairs including THIS currency. E.g. Currency (15) = JPY the response could be USD/JPY; BTC/JPY and ETH/JPY

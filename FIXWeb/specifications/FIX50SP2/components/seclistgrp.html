146 NoRelatedSym N Specifies the number of repeating symbols (instruments) specified
Instrument N The Instrument component block contains all the fields commonly used to describe a security or instrument. Typically the data elements in this component block are considered the static data of a security, data that may be commonly found in a security master database. The Instrument component block can be used to describe any asset type supported by FIX.
55 Symbol N Symbol: common, "human understood" representation of the Instrument. E.g. BTCUSD or XRPJPY
167 SecurityType N Indicates type of security. Security type enumerations are grouped by Product(460) field value. NOTE: Additional values may be used by mutual agreement of the counterparties.
969 MinPriceIncrement N Minimum price increase supported by system, ECN for a given Instrument
15 Currency N Identifies currency used for Qty. Absence of this field is interpreted as the default (base or left side of the pair). It is recommended that systems provide this currency value to avoid confusion.
Can be used to get all Currency Pairs including THIS currency. E.g. Currency (15) = JPY the response could be USD/JPY; BTC/JPY and ETH/JPY
